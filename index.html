4. Propager les incertitudes avec Monte-Carlo¶

Retour la table des matires

On dispose :

  • d'une grandeur $x$ dont la valeur mesure est $x_m$ associe son incertitude-type $u(x)$
  • d'une grandeur $y$ dont la valeur mesure est $y_m$ associe son incertitude-type $u(y)$.

On veut calculer une grandeur $z=f(x,y)$ et estimer l'incertitude-type sur cette grandeur par un algorithme de Monte-Carlo.

La dmarche va se faire en trois temps :

  1. On va gnrer alatoirement des tableaux de valeurs de $x$ et $y$, de distribution uniforme, et cohrentes avec les valeurs des incertitudes-type (c'est dire que, pour $x$, les valeurs sont uniformment rparties dans l'intervalle $\left[x-\sqrt{3}\times u(x) ; x+\sqrt{3}\times u(x)\right]$). Ces listes sont appeles x et y dans la suite.
  2. Pour chaque couple x[i],y[i], on calcule la valeur z[i]=f(x[i],y[i])
  3. L'incertitude-type sur $z$ est l'cart-type du tableau de valeurs de $z$ qui vient d'tre calcul.
In [11]:
import random as rd
import numpy as np
from math import sqrt
import matplotlib.pyplot as plt

def uniforme(m,u,N): # va renvoyer un tableau de N nombres alatoire correspondant  une distribution uniforme 
# centre sur m et d'incertitude-type u
    x=[]
    a=m-sqrt(3)*u
    b=m+sqrt(3)*u
    for i in range(N):
        x.append(rd.uniform(a,b)) # la fonction rd.uniform renvoie un nombre alatoire entre a et b (densit uniforme)
    return np.array(x)

## gnration des tableaux x et y

xm= 5.3 # on rentre ici les donnes issues de l'exprience (grandeur mesure et incertitude-type)
u_x=0.7

ym= 2.1
u_y=0.4

x=uniforme(xm,u_x,10000)
y=uniforme(ym,u_y,10000)

## dfinition de la fonction f

def f(x,y):
    return 3*x**2*y

## calcul de la liste des z

z=f(x,y)

## trac des histogrammes et dtermination de l'incertitude type sur z

plt.figure()

plt.subplot(311)
plt.hist(x,bins=100)
plt.title("Histogramme de x")

plt.subplot(312)

plt.subplot(313)
plt.hist(z,bins=100)

5. Evaluer l'incertitude-type sur un paramètre inconnu avec Monte-Carlo¶

Retour la table des matires

A l'issue d'une modlisation/ajustement d'une courbe avec curve_fit, on peut obtenir la valeur d'un paramtre inconnu ($g$ dans l'exemple plus haut, relire absolument avant, associ une incertitude-type $u(g)$. Cependant, nous ne savons pas comment est dtermine cette incertitude-type et on aimerait la dterminer nous-mme pas un algorithme de type Monte-Carlo.

Comment estimer lincertitude-type sur les paramtres dune fonction $f$ telle que $y=f(x)$ o $y$ et $x$ sont des grandeurs mesures ?

On dispose :

  • dun tableau lx des valeurs de $x$ mesures, de longueur n
  • dun tableau lux des valeurs de $u(x)$ mesures, de longueur n
  • dun tableau ly des valeurs de $y$ mesures, de longueur n
  • dun tableau lux des valeurs de $u(y)$ mesures, de longueur n

L'algorithe consiste en :

Faire $N$ fois :

  • 1) Gnration des tableaux simuls lx_simu et ly_simu avec une distribution alatoire : $$\forall i \mbox{ lx_simu[i] est une valeur alatoire tire uniformment dans }$$ $$\left[lx[i]-\sqrt{3}\times lux[i];lx[i]+\sqrt{3}\times lux[i]\right]$$ On fait de mme pour le tableau des $y$.

  • 2) Estimation des paramtres inconnus l'aide de curve_fit partir de lx_simu et ly_simu. C'est la procdure classique prsente dans le paragraphe "Raliser une modlisation sous Python".

  • 3) Stockage des paramtres estims dans une liste qui grandit chaque itration.

Aprs les $N$ tapes : Observer lhistogramme du(des) paramtre(s) estim(s) et calculer lcart-type associ qui donne lincertitude type sur le paramtre.

import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
from scipy.optimize import curve_fit # c'est la fonction qui ralise l'ajustement
from math import sqrt
import random as rd

mesdonnees = pd.read_excel("determinationg.xls")
deltat=mesdonnees.values[:,0] 
u_deltat=mesdonnees.values[:,1]
h=mesdonnees.values[:,2]
u_h=mesdonnees.values[:,3]

# on redfinit la fonction de modlisation

def f(x,g):
    return np.sqrt(2*x/g)

# on lance l'algorithme de Monte Carlo
liste_g=[]
N=10000
deltat_simu=np.zeros((len(deltat)))
h_simu=np.zeros((len(deltat)))
for k in range(N):
    # tape 1 : on gnre les tableaux simuls
    for i in range(len(deltat)):
        deltat_simu[i]=rd.uniform(deltat[i]-sqrt(3)*u_deltat[i],deltat[i]+sqrt(3)*u_deltat[i])
        h_simu[i]=rd.uniform(h[i]-sqrt(3)*u_h[i],h[i]+sqrt(3)*u_h[i])
    # tape 2 : on estime les paramtres inconnus
    x=h_simu
    y=deltat_simu
    p_estimation,p_err=curve_fit(f, x, y) 
    # tape 3 : on remplit la liste des g estims
    liste_g.append(p_estimation[0])

gm=np.mean(np.array(liste_g))
ug=np.std(np.array(liste_g))

#tracs 
plt.figure()
plt.title("Histogramme de g")
plt.hist(liste_g,bins=100)
plt.text(11,250,"g = "+str(gm)[0:5]+" m/s")
plt.text(11,200,"u(g) = "+str(ug)[0:4]+" m/s")       
plt.show()

7. Résoudre une équation différentielle¶

Retour la table des matires

La rsolution d'quations diffrentielles peut se faire efficacement l'aide de la fonction odeint du module scipy.integrate. Pour l'utiliser, il faut crire le problme rsoudre sous la forme : $$ \frac{dY}{dt}=F(Y,t)$$ $$Y(0)=Y_0$$

Si cette rsolution semble a priori tre uniquement celle d'une quation d'ordre 1, elle permet en fait de rsoudre une quation d'ordre $N$ : il suffit alors que $Y$ soit un vecteur $N$ lments. Nous l'tudierons dans les exemples.

A chaque fois, il faudra donc:

  1. Mettre le problme sous la forme demande et en dduire la fonction $F$
  2. Dfinir la fonction $F$ sous Python
  3. Utiliser y=odeint(F,y0,t,args=()) o args correspond aux arguments de $F$ qui ne sont ni $Y$ ni $t$ (et correspondent des paramtres du problme par exemple). t est le tableau numpy correspondant aux instants o l'on cherche $Y$.

1er exemple simple (et inutile)¶

On cherche dterminer la tension $s$ aux bornes d'un condensateur lors de la charge d'un circuit $RC$, rgi par l'quation diffrentielle : $$s+\tau \frac{ds}{dt}=E$$ $$s(0)=0$$

Le problme se met donc sous la forme : $$\frac{ds}{dt}=\frac{E-s}{\tau}=F(s,t)$$ $$s(0)=0$$

Dfinissons $F$ :

In [13]:
def F(s,t,E,tau): # on met les paramtres du problme (E,tau) en dernier
    return (E-s)/tau

Excutons la fonction odeint et traons le rsultat :

2ème exemple : équation non linéaire d'ordre 2 - le pendule simple¶

On s'intresse dsormais l'quation non linaire d'ordre 2 modlisation les oscillations d'un pendule de longueur $l$, de masse $m$ et soumis une force de frottement fluide de coefficient $h$ : $$\frac{d^2 \theta}{dt^2}+\frac{h}{m} \frac{d\theta}{dt}+\frac{g}{l}\sin(\theta)=0$$ On poser les conditions initiales suivantes : $$\theta(t=0)=\theta_0$$ $$\frac{d\theta}{dt}(t=0)=\omega_0$$ Pour se ramener au problme d'ordre 1 rsolu par odeint, on va poser le vecteur : $$Y= \begin{bmatrix} \theta \\ \dot{\theta}\end{bmatrix}$$ On dduit : $$\dot{Y}= \begin{bmatrix} \dot{\theta} \\ \ddot{\theta} \end{bmatrix}=\begin{bmatrix} \dot{\theta} \\ -\frac{h}{m} \dot{\theta}-\frac{g}{l}\sin(\theta) \end{bmatrix} = F(Y,t)$$ Dfinissons $F$ sous Python :

def F(Y,t,h,m,g,l):
    return Y[1],-h/m*Y[1]-g/l*sin(Y[0])

3ème exemple : l'oscillateur à Pont de Wien¶

Dans le TP sur l'oscillateur quasi-sinusodal Pont de Wien, on montre que la tension de sortie de l'amplificateur oprationnel $s$ tait rgie par le jeu d'quations suivant :

  • Si $|s| < V_{sat}$ : $$ s + \frac{1-\frac{K}{3}}{Q\omega_0} \dot{s} + \frac{1}{\omega_0^2}\ddot{s} = 0$$
  • sinon : $$ s + \frac{1}{Q\omega_0} \dot{s} + \frac{1}{\omega_0^2}\ddot{s} = 0$$

o :

  • $K$ est le coefficient de l'amplificateur utilis
  • $Q$ et $\omgea_0$ sont respectivement le facteur de qualit et la pulsation caractristique du filtre passe-bande de Wien.

Voyons ci-dessous comment rsoudre ce problme, en posant : $$Y= \begin{bmatrix} s \\ \dot{s}\end{bmatrix}$$ et $$\dot{Y}=F(Y,t)$$

import numpy as np
from scipy.integrate import odeint
import matplotlib.pyplot as plt

#dfinissons F

def F(Y,t,Q,w0,K,Vsat):
    if abs(Y[0])<Vsat:
        d2y=-w0**2*Y[0]-w0/Q*(1-K/3)*Y[1]
    else :
        d2y=-w0**2*Y[0]-w0/Q*Y[1]
    return Y[1],d2y
